## Phase 1: Split Hono API Server from React Router SSR Client

### Overview

This PR separates the Documenso monolith into two independently runnable
Node.js processes as the first step toward a full 3-tier architecture.
Previously, a single process on port 3000 handled everything — Hono API
routes, React Router SSR, and static file serving. After this change, the
API and client servers are decoupled and can be started, built, and deployed
independently.

---

### Architecture After This PR

Port 3000 — Client Server (React Router SSR)
Serves the React UI and static assets
Proxies all /api/\* browser requests → Port 3001

Port 3001 — API Server (Hono)
Handles all /api/\* routes (auth, tRPC, REST, files, jobs, AI)
Direct access to the database via Prisma

> Note: Both servers still share the same environment variables and database
> access in this phase. Full tier isolation comes in Phases 3 and 4.

---

### Files Changed

**`server/main-api.ts`** — New file
Standalone entry point for the Hono API server. Imports the existing Hono
app from `server/router.ts` and starts it on `SERVER_PORT` (default 3001).
Used in development via Rollup watch + Node.js watch.

**`server/main-client.js`** — New file
Standalone entry point for the React Router SSR client server. Runs on
`PORT` (default 3000). Proxies all `/api/*` browser requests to the API
server, serves static build assets, and handles React Router SSR. Written
as plain JavaScript (not TypeScript) because it imports `./index.js` — the
React Router build artifact — which does not exist at Rollup compile time.

**`rollup.config.mjs`** — Modified
Changed the input entry point from `server/router.ts` to `server/main-api.ts`.
Rollup now builds the full API server (including router.ts as a preserved
module dependency) into `build/server/hono/`.

**`vite.config.ts`** — Modified
Removed the `hono-react-router-adapter/vite` server adapter plugin, which
previously ran Hono inside the Vite dev server. Replaced with Vite's
built-in `server.proxy` configuration that forwards all `/api/*` requests
from the Vite dev server (port 3000) to the Hono API server (port 3001).

**`apps/remix/package.json`** — Modified
Added new scripts to support running both servers independently in
development and production. Added `concurrently` as a dev dependency to
run both dev servers in a single terminal. Key scripts added:

- `dev:api` — builds the API server once then starts rebuild watch + node watch
- `dev:api:start` — runs Rollup watch and Node watch concurrently
- `dev:api:rebuild` — Rollup in watch mode for API source changes
- `dev:api:serve` — Node.js in watch mode, restarts on Rollup output changes
- `dev:client` — Vite dev server for React Router (unchanged behavior)
- `dev` — updated to run both `dev:api` and `dev:client` concurrently
- `start:api` — runs the built API server in production
- `start:client` — runs the built client server in production

**`apps/remix/tsconfig.json`** — Modified
Added wildcard path entries (`@documenso/lib/*`, `@documenso/auth/*`, etc.)
for all workspace packages. The original paths only mapped root-level
imports. Without wildcard entries, tsx and the TypeScript compiler could not
resolve sub-path imports like `@documenso/lib/server-only/...` to their
TypeScript source files, causing ESM named export errors at runtime.

### New Environment Variables

| Variable      | Used By       | Default                 | Purpose                               |
| ------------- | ------------- | ----------------------- | ------------------------------------- |
| `SERVER_PORT` | API server    | `3001`                  | Port the Hono API server listens on   |
| `SERVER_URL`  | Client server | `http://localhost:3001` | Where the client proxies `/api/*`     |
| `CLIENT_URL`  | API server    | `http://localhost:3000` | Reserved for CORS in a future phase   |
| `PORT`        | Client server | `3000`                  | Port the client SSR server listens on |

---

### Development Workflow

```bash
# Start both servers (API on 3001, Client on 3000)
npm run dev

# Or start them independently
npm run dev:api     # API server only
npm run dev:client  # Client server only
```
